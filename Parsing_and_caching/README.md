Обработка журналов событий

Суть задачи в следующем:
На входе дан путь к каталогу с файлами. Файлы являются журналами событий (лог файлами). 
Существует правило именования файлов: {{service_name}}_{{ date }}.log
где service_name - имя сервиса, события по которому попадают в журнал, а date - дата в формате YYYYMMDDHH, например 2026032420. Таким образом, каждый файл содержит события за определенный час. События в файле упорядочены по времени.
В каждом файле построчно с разделителем строк b'\n'  пишутся логи в json формате.
Поля json объекта:
datetime - дата события (обязательно попадает в час из имени файла)
event_type - уровень (ERROR, INFO, DEBUG)
message - текст сообщения
params - словарь для интерполяции переменных в текст сообщения вида {“some_var”: “some_value”}


Нужно написать класс, в который в методе __init__ передается путь к папке с файлами, реализующий различные методы по работе с журналами событий.

При реализации класса считаем, что новые файлы только появляются, старые не меняются
