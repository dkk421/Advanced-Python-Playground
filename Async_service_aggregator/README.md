 асинхронный CLI-агрегатор, который опрашивает набор виртуальных сервисов, обрабатывает сбои, ограничивает нагрузку и формирует детальный отчёт.

Работа по шагам.

 Шаг 1. Базовый асинхронный опрос с таймаутами
Реализована функция-симуляция simulate_service(name: str, delay: float) -> dict, которая:
Ждёт delay секунд (asyncio.sleep)
Возвращает {"name": name, "status": "ok", "latency": delay}
Написана асинхронная функция check_services(configs: list[dict], timeout_per_service: float), которая запускает опрос всех сервисов конкурентно.

asyncio.timeout(), каждый вызов обернут в соответствующий контекстный менеджер. При превышении timeout_per_service сервис должен получать статус timeout.
Функция возвращает список результатов в порядке вызова.

Параллельный запуск корутин через `asyncio.TaskGroup`

ШАГ 2: Retry с экспоненциальной задержкой

Логика опроса обернута в механизм повторных попыток.

Цикл `for attempt in range(MAX_RETRIES + 1)` даёт ровно 4 попытки (первая + 3 ретрая)

Обработка исключений точечная: перехватываются только `asyncio.TimeoutError` и `ConnectionError`. Нет риска скрыть непредвиденные ошибки.

Счётчик попыток (`attempts`) корректно отражает реальное количество вызовов для каждого сервиса, включая финальную успешную или провальную попытку

ШАГ 3: Ограничение конкурентности

Добавлен asyncio.Semaphore, ограничивающий количество одновременных запросов до 2.

Семафор размещён снаружи retry-цикла, поэтому сервис удерживает слот конкурентности на протяжении всех повторных попыток. Это гарантирует, что общее число одновременно выполняющихся сервисов никогда не превышает установленный лимит, даже при retry.

Удержание семафора на время `asyncio.sleep` между ретраями - это осознанный выбор, который немного снижает параллелизм, но упрощает логику.
